# weak-llm-playbook

![tests](https://github.com/mmzz164/weak-llm-playbook/actions/workflows/tests.yml/badge.svg)

弱い/安価なLLMへのコーディング・テキストタスク委譲を、勘ではなく**測定**で運用する。

English version: [README.md](README.md)

## 実際にあった話から

ローカルモデルに注文数量の抽出を頼んだ。原文には「3〜5個」とある。モデルが抽出したのは
**4**——勝手に発明された中央値——しかも5回中5回。エラーも警告もなく、完全に安定。
そのまま流していれば、下流のレコード全部が静かに間違っていた。

弱いLLMは「難しいから」失敗するのではない。失敗するのは**書いていないこと**であり、
危険なのは安定した失敗のほう(揺れないので、永遠に気づけない)。このツール群はそれを
測定可能にし、機械的に直す: 同じ指示をK回実行して同じ入力で結果を比べ、**割れた所=
あなたの書き忘れ**。ピン留めして再測定し、消えたことまで検証する。

## まず10秒で試す — モデル・GPU・APIキー不要

```bash
python3 demo/demo.py
```

決定論モックを相手に本物のパイプラインが走る: 曖昧さの検出(「上位n件」— 値順?先頭n個?)
→ ピン留め → 検証(`holes: 2 → 0`)→ 実行 → 再生照合まで。

## 自分のエンドポイントで使う

依存ゼロ(Python標準ライブラリのみ)。OpenAI互換エンドポイント
(vLLM / llama.cpp / ollama / OpenAI API)とAnthropic APIに対応。

```bash
export PROBE_BASE=http://localhost:8000       # ollamaなら http://localhost:11434

# 中核ツール — ドラフト指示の穴を見つけ、再現性を検証済みの修正版プロンプトを得る:
python3 tools/spec_holes.py draft.txt inputs.json --fix

# モデルごとに1回 — 既定の癖を測定し、委譲ガイド1枚に集約:
python3 tools/default_probe.py $PROBE_BASE 5
python3 tools/model_card.py --glob 'profiles/*.json' -o card.md

# 全部入り1コマンド(振り分け・「このページをレビューして」等の自由文タスクへの
# 影のJSON契約・ゲート・検証済み実行):
python3 tools/selffix.py draft.txt [inputs.json] [--run]

# ツールを使うタスク(MCPでトラッカー検索など)は使い捨てエージェントセッションで
# 実行 — エージェントCLI(`claude` かローカルラッパー)が必要:
python3 tools/run_agent.py task.txt --fix
```

**持ち出し**: 中核は標準ライブラリのみの2ファイル——`tools/spec_holes.py` +
`tools/llm_client.py`。どのプロジェクトにもそのまま置ける。

CLIリファレンス・パック形式・比較ポリシーの詳細:
[docs/USAGE.ja.md](docs/USAGE.ja.md)。

## 設計の元になった発見

- 明示した規則は、反直感なものでも守られる。失敗の原因は常に曖昧さか書き漏らしで、
  「難しさ」ではない。
- 一番効くのは安定した既定: 「3〜5個」は5回中5回、**発明された中央値4**として抽出される。
- 既定も、指示遵守そのものも、**プロンプトの言語**に依存する: ある3.8Bモデルは
  「setを使うな」を日本語では40%の確率で破るが、英語では一度も破らない。
  委譲に使う言語で測ること。
- モデルを替えると挙動が黙って変わる(欠損フィールド: あるモデルは`null`、別のモデルは
  値を捏造)。`--diff` が機械的に捕まえる。
- テキストの「止まれ」という指示は、エージェントの親切心に**2戦2敗**だった。
  手順はコードに、権限はハーネスに——このツール群自体がその原則で作られている。

## 注意

- プローブは**生成コードを実行する**。信頼できないエンドポイントにはサンドボックスを。
- プロファイルはモデル×量子化×プロンプト言語ごと。どれかが変わったら再測定
  (それを自動化するのが `--assert`)。
- 自由文の**質**(良い要約か)はここでは測れない。測るのは構造・件数・顔ぶれ・再現性。
