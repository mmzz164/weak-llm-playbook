# weak-llm-run — 正常化して、弱LLMに実行させ、検証して返す

引数 = ドラフト(+抽出タスクなら対象データのパス)。
成果物 = **検証済みの実行結果**(コードファイル or 抽出JSON)+検証レポート。

## 手順

1〜7. **正常化**: [[weak-llm-fix]] の手順1〜7と同一(そちらのSKILL.mdに従う)。
   fixed.txt(完成プロンプト)ができた状態から続ける。

8. **実行**: `python3 <scripts>/run_prompt.py fixed.txt [--input <対象データ>]`
   - コードタスク: 出力からコードを取り出してファイルに保存する。
   - 抽出タスク: 対象データ(文書)ごとに `--input` で実行し、JSONを収集する。
     対象データが渡されていなければプローブ入力を対象として使い、その旨を明記する。

9. **独立検証(必須。弱LLMの自己申告は信じない)**:
   - コード: **自分で**隠しテストを書いて実行する(境界値中心に5件以上。fixed.txt の
     ピン行がそのままテストケースになる)。失敗したら外した境界を fixed.txt に1行足して
     再投下(最大2回)。それでも失敗するなら委譲不適として経緯ごと報告する。
   - 抽出: 全出力をパースし、(a)ピン留めした形式に従っているか (b)欠損の扱いが指示どおりか
     (c)値が入力文書に実在するか(捏造検知)を確認する。

10. **報告**: 成果物のパス / 使ったプロンプト(fixed.txt)のパス / 検証結果
    (何を当てて何件通ったか)/ 要確認の点。以上を簡潔に。

## 注意
- 生成コードの実行とテストはこのセッション内で行う。危険な操作(ファイル削除・ネットワーク)を
  含むコードが返ってきたら実行せずユーザーに見せる。
- 再投下は2回まで。ループさせない。
